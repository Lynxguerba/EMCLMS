# Notifications and Logs

This document outlines the specific information recorded and transmitted by the system's Notification and Logging mechanisms.

## 1. Notifications (System to User)
Notifications are triggered by administrative or instructional actions and alert users to events that affect them. They are stored in the `Notification` model.

### Recorded Information
Each notification captures:
- **Recipient (`user`)**: The specific user receiving the alert.
- **Title**: A short summary of the event.
- **Message**: Detailed information or instructions.
- **Status**: Whether the notification is `Unread` or `Read`.
- **Timestamp (`created_at`)**: When the notification was issued.

### Event Triggers and Content
The table below lists the specific titles and messages generated by the system:

| Event                       | Recipient      | Title                   | Message Example                                                                                             |
| :-------------------------- | :------------- | :---------------------- | :---------------------------------------------------------------------------------------------------------- |
| **Course Creation**         | Instructor     | `New Course Assignment` | "You have been assigned as the instructor for the new course: [Course Code] ([Course Title])."              |
| **Instructor Reassignment** | New Instructor | `New Course Assignment` | "You have been assigned as the new instructor for the course: [Course Code] ([Course Title])."              |
| **Course Deletion**         | Instructor     | `Course Deletion Notice` | "The course '[Course Title]'... has been deleted by an administrator."                                      |
| **Student Enrollment**      | Student        | `New Course Enrollment` | "You have been enrolled in the course: [Course Code] ([Course Title])."                                     |
| **Student Unenrollment**    | Student        | `Course Unenrollment`   | "You have been unenrolled from the course: [Course Code] ([Course Title])."                                 |
| **Grading**                 | Student        | `Submission Graded`     | "Your submission for '[Assignment Title]' has been graded."                                                 |
| **Accounting Update**       | Student        | `Account Update: [Type]`| "A new [Type] of ₱[Amount] for [Category] has been recorded to your account."                               |
| **Credit Application**      | Student        | `Account Update: Payment`| "Successfully applied ₱[Amount] from your available credits to your outstanding balance."                   |
| **Account Charge**          | Student        | `Account Charge Applied`| "A charge of ₱[Amount] for [Category] has been applied to your ledger."                                     |

## 2. External Email Notifications
For sensitive security actions and account lifecycle events, the system transmits emails via the **Brevo (formerly Sendinblue) API**.

### Account Approval & Creation
When a registration request is approved or a user is manually created by an Administrator, an email is sent containing the user's login credentials.

- **Trigger:** `POST api/admin/create-user/`
- **Sender:** Configured via `BREVO_SENDER_EMAIL`.
- **Content:** Includes the user's email address and the manually assigned password.
- **Instruction:** Users are advised to change their password immediately upon their first login.

## 3. User Activity Logs
Logs track the activity history of users within the platform, primarily triggered by actions on the frontend. They are stored separately in the `StudentLog` and `InstructorLog` models.

### Recorded Information
Each log entry captures:
- **Actor (`student` or `instructor`)**: The user performing the action.
- **Message**: A text description of the activity sent from the frontend.
- **Timestamp (`timestamp`)**: The exact date and time (stored in UTC) when the action occurred.

### Logged Activities
The frontend records session management activities via the `/api/student/post/add-student-log/` and `/api/instructor/post/add-instructor-log/` endpoints:

| User Type      | Action | Logged Message  | Source Component  |
| :------------- | :----- | :-------------- | :---------------- |
| **Student**    | Login  | `"Logged in"`   | `LoginPage.tsx`   |
| **Student**    | Logout | `"Logged out"`  | `Sidebar.tsx`     |
| **Instructor** | Login  | `"Logged in"`   | `LoginPage.tsx`   |
| **Instructor** | Logout | `"Logged out"`  | `Sidebar.tsx`     |

> **Note:** The logging system is designed to accept any string message. Additional user interactions (e.g., "Viewed Lesson", "Started Quiz") can be easily recorded in the future by adding `axios.post` calls to the respective log endpoints in other frontend components.
